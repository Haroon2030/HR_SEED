# خارطة طريق تطوير نظام الموارد البشرية

مستند مرجعي للمراحل التالية بعد استقرار الأمان والمرحلة 0 (توثيق، CI، Spectacular، تنظيف صلاحيات الحضور).

## تم إنجازه (مرجع سريع)

- أمان API وRBAC، عزل فروع الموظفين، media محمي، JWT + blacklist، توثيق OpenAPI، `/health/`، CI على GitHub.
- هجرة تعطيل صلاحيات `attendance.*` الوهمية في قاعدة البيانات.

## المرحلة 2 — قيمة أعمال

اختر واحداً أو اثنين حسب أولوية المنشأة:

1. **تنبيهات انتهاء الوثائق** (جواز، إقامة، تأمين، عقد) مع لوحة وتذكير بريدي.  
   - *منجز:* حقول تواريخ الإقامة والتأمين الطبي وانتهاء العقد المخطط على الموظف؛ تجميعها في `collect_expiring_documents`، النماذج، العرض، التصدير، واللوحة/الأمر/البريد كما سبق.  
   - *لاحقاً:* تقارير compliance إضافية، تذكيرات أدق حسب الفرع.
2. **تصدير Excel/PDF** لأهم التقارير من مركز التقارير.
3. **وحدة حضور حقيقية** (سياسات دوام + تسجيل يدوي ثم تكامل لاحق).
4. **بوابة موظف مبسطة** (رصيد إجازة، إشعارات، طلب إجازة) بعد توسيع API إن لزم.

## المرحلة 3 — توسع

- API كامل للموبايل (موظفين، موافقات، مسير، إشعارات).
- أداء: Celery للمهام الثقيلة (تقارير، بريد جماعي).
- SaaS متعدد العملاء فقط بعد قرار تجاري واضح (عزل `tenant` على البيانات).
